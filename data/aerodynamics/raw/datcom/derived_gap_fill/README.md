# Derived DATCOM Gap-Fill Candidates

This directory stores Mach-dependent candidate coefficients that are not
printed directly by the current Digital DATCOM static/damping or control
outputs, but are needed by the active aerodynamic derivative interface.

The table is generated by:

```sh
tools/build_datcom_gap_fill_candidates.py \
    data/aerodynamics/raw/datcom/mach_grid/b777_like_derivative_datcom_candidate_mach_grid.csv \
    data/aerodynamics/raw/datcom/control_grid/b777_like_control_derivative_datcom_candidate_mach_grid.csv \
    data/aerodynamics/raw/datcom/derived_gap_fill/b777_like_derivative_gap_fill_candidate_mach_grid.csv \
    --report data/aerodynamics/raw/datcom/derived_gap_fill/b777_like_derivative_gap_fill_candidate_mach_grid_report.txt
```

The generated rows use source id:

```text
DIGITAL_DATCOM_B777_LIKE_DERIVED_GAP_FILL_CANDIDATE_V0
```

These rows are not direct DATCOM printouts. They are low-confidence candidate
values derived from the Digital DATCOM Mach-grid candidates, the
elevator/aileron control-grid candidates and the committed B777-like geometry.
They fill `CD_beta`, `CY_r`, `CD_delta_a`, `CY_delta_r`, `Cl_delta_r`,
`Cn_delta_r` and `CD_delta_r` so that promotion gates can check the full active
derivative interface without silently falling back to seed values.

The active plant must not consume these values until the project promotion
gates are satisfied. The clean-cruise longitudinal trim and restricted
longitudinal modal gates are already documented separately.
